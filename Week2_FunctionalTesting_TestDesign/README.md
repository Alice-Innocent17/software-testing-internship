Week 2 - Functional Testing and Test Design

This folder contains the tasks completed during week 2 of the software testing internship. The focus of this week was on understanding and applying software testing techniques used for designing effective test cases.

TASKS COMPLETED:

1. Test Scenario and Test Case Writing (Instagram)
   Test scenarios and detailed test cases were created for the Instagram application to validate different functionalities such as login, signup, post interactions, and user actions. The scenarios provide a high-level view of what to test, while the test cases describe step-by-step verification of each functionality.
   
2. Boundary Value Analysis (BVA)
   Boundary Value Analysis was applied to validate input fields by testing values at the boundaries of the specified ranges. Test cases were created for:
   - Age input field (18–60)
   - Password length field (8–16)
   - Amount field (1–10,000)

3. Equivalence Partitioning (EP)
   Equivalence Partitioning technique was used to divide input data into valid and invalid partitions. Representative test cases were created for:
   - Email field
   - Phone number field
   - Date of birth field

 FILES INCLUDED:

- Task4_Instagram_TestCase_TestScenarios.xlsx – Contains test scenarios and test cases for the selected application.
- Task5_BVA.xlsx – Boundary Value Analysis tables for the given input ranges.
- Task6_EP_TestCases.docx – Document containing Equivalence Partitioning test cases.

LEARNING OUTCOME:

Through these tasks, the following concepts were practiced:
- Writing effective test scenarios
- Designing detailed test cases
- Applying Boundary Value Analysis (BVA)
- Applying Equivalence Partitioning (EP)
* Documenting test artifacts in structured formats
