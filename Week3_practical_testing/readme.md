WEEK 3 - PRACTICAL TESTING AND API TESTING

This folder contains the tasks completed during Week 3 of the software testing internship. The focus of this week was on performing real-time UI/UX testing, API testing using Postman, and preparing a test summary report.

TASKS COMPLETED:
1. UI/UX Testing (BookMyShow)

BookMyShow website was tested to evaluate the user interface and user experience. The testing focused on identifying visual and usability issues that affect user interaction.

The following aspects were analyzed:
   - Layout issues
   - Responsiveness across different screen sizes
   - Color and spacing consistency
   - Font consistency

Screenshots were captured to highlight the identified issues and included in the report.

2. API Testing Using Postman

API testing was performed using Postman on a public API.

The following operations were tested:
  - GET – Retrieving data from the server
  - POST – Creating new data
  - PUT – Updating existing data
  - DELETE – Removing data

Each request and response was validated, and screenshots were captured for documentation. The Postman collection was also exported in JSON format.

3. Test Summary Report

A test summary report was prepared based on the testing performed on the selected website.

The report includes:
   - Scope of testing
   - Objectives
   - Test scenarios summary
   - Defect summary
   - Recommendations for improvement

This document provides an overall view of the testing process and findings.

- Task8_API_Screenshots/ – Screenshots of API requests and responses

- Task9_test_summary_report.pdf – Final test summary report

LEARNING OUTCOME:

Through these tasks, the following concepts were practiced:
   - Performing real-time UI/UX testing on live websites
   - Identifying usability and design issues
   - Understanding API testing concepts
   - Using Postman for executing API requests
   - Documenting test results in professional reports
